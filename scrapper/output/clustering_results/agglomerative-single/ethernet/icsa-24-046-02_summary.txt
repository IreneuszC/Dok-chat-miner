Siemens has identified vulnerabilities in their products, including an infinite loop in OPC UA .NET Standard Stack 1.04.368 and issues with SSL_check_chain() function and X.509 GeneralName type in OpenSSL. These vulnerabilities could allow remote attackers to cause denial-of-service attacks. Siemens has released workarounds and mitigations, and recommends updating to the latest version of SIDIS Prime. No known public exploitation has been reported, but organizations are encouraged to implement cybersecurity strategies and report any suspicious activity to CISA.
